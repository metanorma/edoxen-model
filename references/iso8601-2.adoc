ISO 8601-2 — Date and Time Extensions
=====================================

ISO 8601-2 (2nd edition, 2026) extends ISO 8601 with formal
syntax for time intervals, durations, recurring time intervals,
date/time selection, calculations, and profiles. Edoxen adopts
ISO 8601-2 as the canonical date/time/recurrence standard,
referenced (not duplicated) throughout the model.

| ISO 8601-2 concept | Edoxen model

| §10 Time intervals | `DateRange` (existing) — start + end pair
| §11 Durations | (used inside `DateRange.duration` and `Recurrence.interval`)
| §12 Date/time selection | (not directly modelled — used by adopter-side tooling)
| §13 Recurring time intervals | `Recurrence` (`models/recurrence.lutaml`) — structured, queryable
| §14 Calculations | (not modelled — adopter-side gem provides arithmetic)
| §15 Profiles | `MeetingExtension` (`models/meeting_extension.lutaml`) — adopter-extensible

== Why ISO 8601-2 over iCalendar RRULE

iCalendar (RFC 5545) defines `RRULE` for recurrence, but Edoxen
uses ISO 8601-2 §13 instead because:

1. ISO 8601-2 is the international standard (ISO/TC 154/WG 5);
   iCalendar is an IETF-specific format.
2. ISO 8601-2 §13 syntax is more compact and unambiguous.
3. ISO 8601-2 §15 profiles concept is reused by Edoxen for
   adopter customization (`MeetingExtension`).
4. ISO 8601-2 §14 calculations provide arithmetic semantics that
   iCalendar lacks.

The `Recurrence` class is structured (each BYxxx part is its own
field) so it's queryable. The gem provides a `to_iso8601_2` /
`from_iso8601_2` round-trip adapter (declared via lutaml-model
mapping, not hand-rolled `to_h`).
