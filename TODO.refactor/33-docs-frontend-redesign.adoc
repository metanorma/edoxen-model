= 33 — Docs: Frontend redesign with /frontend-design

== Why

User: "Make sure we have good specs throughout" + "update rendering code using /frontend-design".

The docs site (VitePress) needs a frontend redesign to reflect the new model visually and improve UX.

== Approach

1. Invoke `/frontend-design` skill
2. Audit existing `.vitepress/theme/`
3. Redesign:
   - Homepage (`index.md`)
   - Sidebar nav (config.ts)
   - Page templates
   - Code block styling
   - State-machine diagrams (SVG)
   - Entity relationship diagrams (SVG or Mermaid)

== Specific design elements

- **State machine diagrams** for Motion/Voting/Decision/Topic status flows
- **Entity relationship diagrams** showing how Meeting/Decision/Motion/Voting/Topic interconnect
- **Profile extension visual** showing how `MeetingExtension` hangs off core entities
- **Sample YAML explorer** (interactive: click entity → highlight related)

== Files to create/modify

- `.vitepress/theme/` (custom theme)
- `.vitepress/components/` (Vue components)
- `docs/.assets/diagrams/` (SVG diagrams)

== Acceptance criteria

- New theme is distinctive (not generic AI)
- All diagrams render correctly
- Mobile-responsive
- Accessibility: WCAG 2.1 AA
