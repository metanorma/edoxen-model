= 19 — Create sample YAML demonstrating broadened scope

== Why

A concrete sample proves the model works end-to-end. Best
candidate: HK LegCo sitting (because it exercises Topic,
Motion, Voting, Decision, MeetingComponent, profile extensions
all in one document).

== Files to create

- `samples/legco-sitting-2024-01-15.yaml` (or similar)
- `samples/oiml-ciml-56.yaml` (existing OIML sample, migrated)
- `samples/ietf-113.yaml` (multi-track technical meeting)
- `samples/hybrid-board-meeting.yaml` (corporate, polymorphic venues)

== Sample 1: HK LegCo (demonstrates broad coverage)

Should include:
- Meeting with series_ref to MeetingSeries (8th LegCo)
- Multiple venues (Chamber physical + video bridge virtual)
- Agenda with items pointing to Topics
- Topics with TopicDocuments (bills via profile extension)
- Motions with multiple seconders
- Votings with division method, ayes/noes counts, casting_vote
- Decisions (kind=resolution) resulting from carried motions
- MeetingComponents (debates, question_time, address)
- Extensions (profile="legco" for summoning-bells, etc.)

== Sample 2: OIML CIML (demonstrates standards-body backward compat)

Should include:
- Meeting with series_ref to MeetingSeries (CIML)
- Single physical venue (UNLOCODE: FRPAR)
- Officers (chair, secretary)
- Agenda with numbered items
- Decisions (kind=resolution) — pre-existing OIML semantics
- Motions (mostly unanimous_consent)
- Votings with show_of_hands method
- Minutes sections

== Sample 3: IETF (demonstrates multi-track technical)

Should include:
- MeetingSeries (IETF meeting N)
- Multiple physical venues (UNLOCODEs for host city)
- Multiple virtual venues (Zoom, Meetecho, audio stream)
- MeetingComponents: plenary_session, wg_session, bof, break
- Topics (Internet-Drafts as TopicDocuments)
- Decisions (mostly kind=recommendation — IETF doesn't pass binding resolutions)

== Sample 4: Hybrid corporate board

Should include:
- Single physical venue + Zoom + phone dial-in
- Officers (chair, secretary, treasurer)
- Topics with TopicDocuments (board papers)
- Motions with multiple seconders (some boards require 3)
- Votings with roll_call method
- Decisions (kind=order — board orders)
