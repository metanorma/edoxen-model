= 03 — Motion: procedural act entity

== Why

User (2026-07-03): "A motion is the ability to BRING a decision.
a resolution is a kind of decision. When you say vote on
resolution it means 'vote on a DRAFT resolution', if it is
accepted it becomes a real 'resolution' (decision)."

And: "A Motion does not need ANY document to be raised and
carried upon to a Resolution."

A Motion is a procedural act (someone says "I move that...").
It is distinct from a Decision (the formal outcome) and from
a Document (a TopicDocument that may or may not exist).

Motion is the canonical anchor for Votings. Multiple Votings
can occur on one Motion (unclear voice vote → chair calls
division → second Voting instance).

== Schema

```lutaml
class Motion {
  identifier: String
  urn: String
  text: String                          # "I move that ..."
  mover: Person                         # who proposed
  seconders: Person[0..*]               # OPEN list (0 for consensus bodies,
                                         # 1 in most parliamentary, 3+ in some)
  status: MotionStatus                  # state machine
  introduced_at: DateTime
  proposed_decision: String[0..1]       # URN of Decision being brought
                                         # (the draft, if any)
  resulting_decision: String[0..1]      # URN of adopted Decision
                                         # (filled when status=carried)
  votings: String[0..*]                 # URNs → Meeting.votings
  extensions: MeetingExtension[0..*]
}

enum MotionStatus {
  introduced,         # mover has stated the motion
  seconded,           # one or more seconders obtained
  debating,           # chair has opened debate
  question_put,       # chair has put the question
  voting,             # vote being taken
  carried,            # → resulting_decision URN set
  negatived,
  withdrawn,
  lapsed              # no decision before end of meeting
}
```

== State machine

```
introduced → seconded → debating → question_put → voting
                                                 ├── carried → resulting_decision
                                                 ├── negatived
                                                 ├── withdrawn
                                                 └── lapsed
```
